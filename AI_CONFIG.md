# SmartMeeting 全局 AI 配置

本文件作为仓库内 AI 助手的统一配置入口。

## 1. 语言规则

- 始终使用中文（简体）回复用户。
- 对外说明、步骤、结论均使用中文。
- 代码、命令、路径、接口字段名保持英文原样。

## 2. 响应风格

- 先给结论，再给关键步骤。
- 保持简洁，避免与任务无关的冗长内容。
- 涉及改动时明确说明：改了什么、为什么改、如何验证。

## 3. 开发与改动原则

- 小步迭代，优先增量修改现有文件。
- 非必要不重写整文件、不做大范围重构。
- 不擅自删除用户已有改动。
- 禁止硬编码密钥、令牌、密码等敏感信息。

## 4. 技术栈与目录约定

- 后端：FastAPI（`backend/`）
- 前端：Vue 3 + TypeScript（`frontend/`）
- 数据库：MySQL（`database/`）
- 文档：`docs/`

## 5. 代码规范要点

### Python / FastAPI

- 路由层保持轻量，业务逻辑放在 `backend/app/services/`。
- 使用 Pydantic 做请求/响应校验。
- 公共函数补充类型标注。

### Vue 3 / TypeScript

- 使用 `<script setup lang="ts">` + Composition API。
- 组件命名使用 PascalCase。
- 避免 `any`，异常要有可见反馈。

### MySQL

- 表名与字段名使用 `snake_case`。
- 常规时间字段使用 `created_at`、`updated_at`。

## 6. 推荐实现顺序（MVP）

1. 项目初始化
2. Git 与协作流程
3. 数据库设计
4. 用户系统
5. 会议管理
6. 音频上传
7. Whisper 语音识别
8. AI 会议总结与任务提取
9. 任务管理与前端联调
10. 系统测试

## 7. 质量与验证

- 功能变更需补充或更新测试。
- 无法执行命令时，明确说明原因与替代方案。
- 交付时提供可复现的验证命令与预期结果。
