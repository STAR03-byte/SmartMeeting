## 1 引言

### 1.1 编写目的

本说明书用于介绍 SmartMeeting 系统的运行环境、安装方法和主要功能操作流程，方便老师、同学或项目组成员快速了解并使用本系统。本文档在编写时尽量结合项目当前已经实现的页面、脚本和功能来说明，避免出现与系统实际情况不一致的问题。

### 1.2 软件简介

SmartMeeting 是一个面向会议记录、会议摘要和任务整理的智能会议系统。系统采用 FastAPI + Vue 3 + TypeScript + MySQL 技术栈实现，主要目标是把会议中的音频、转写、摘要和任务集中到同一套系统里进行管理，减少人工整理会议纪要和分发任务的重复工作。

目前系统已经支持的主要功能包括：

- 用户登录与注册；
- 会议创建、查看、筛选和删除；
- 音频上传与会议转写；
- 在线录音与自动转写；
- 会议摘要生成；
- 会议任务提取与管理；
- 团队管理与成员邀请；
- 会议分享；
- AI 助手辅助处理；
- 热词管理。

### 1.3 适用对象

本系统主要适用于以下场景：

- 课程项目小组会议记录；
- 项目组任务分配与跟踪；
- 课程答辩演示；
- 会议纪要整理与共享。

## 2 运行环境与安装说明

### 2.1 运行环境

根据项目当前的启动脚本、README 和联调文档，系统建议运行环境如下表所示。

| 项目 | 建议环境 |
|------|----------|
| 操作系统 | Windows 10/11、Linux、macOS |
| Python | 3.8 及以上 |
| Node.js | 18 及以上 |
| 数据库 | MySQL 8.0 |
| 浏览器 | Chrome、Edge 等现代浏览器 |
| 可选环境 | Docker、Whisper、LLM 服务 |

补充说明如下：

- 在线录音功能依赖浏览器麦克风权限；
- 如果没有配置真实的 LLM 服务，系统仍可运行，但摘要和任务提取会使用回退逻辑；
- 如果没有配置 Whisper，系统也可以运行，但语音识别会使用模拟或回退方式处理。

### 2.2 使用 Docker 启动

在项目根目录执行：

```bash
docker compose up --build
```

启动完成后，默认访问地址如下：

- 前端：`http://localhost:5174`
- 后端：`http://localhost:8000`
- MySQL：`localhost:3307`

这种方式比较适合第一次运行系统，因为前后端和数据库会一起启动，配置相对统一。

### 2.3 本地开发一键启动

在项目根目录执行：

```bash
npm run dev
```

该命令会自动完成以下工作：

- 检查依赖是否已经准备好；
- 如有缺失，自动调用 `scripts/dev/bootstrap.py` 安装依赖；
- 启动后端服务；
- 自动选择前端端口并启动前端。

默认情况下：

- 后端地址为 `http://127.0.0.1:8000`
- 前端地址优先为 `http://127.0.0.1:5173`

### 2.4 手动启动前后端

如果需要分开调试，也可以手动执行以下命令。

后端：

```bash
python -m pip install -r backend/requirements.txt
python -m uvicorn backend.main:app --reload
```

前端：

```bash
npm --prefix frontend install --cache "D:\SmartMeeting\.npm-cache"
npm --prefix frontend run dev
```

如果本地后端不是运行在默认端口，可以先设置：

```bash
set SMARTMEETING_DEV_BACKEND_URL=http://127.0.0.1:8888
```

### 2.5 数据库初始化

第一次搭建数据库时，可以执行：

```bash
mysql -u <user> -p < scripts/db/run_all.sql
```

初始化完成后，可以继续执行：

```bash
mysql -u <user> -p < scripts/db/check_health.sql
```

## 3 系统登录与首页功能

### 3.1 登录与注册

系统登录页路由为：`/login`。

登录页提供“登录”和“注册”两个页签。使用时可以按照下面步骤操作：

1. 打开系统；
2. 如果当前未登录，系统会自动跳转到登录页；
3. 已有账号时，输入用户名和密码后点击“登录”；
4. 如果没有账号，可以切换到“注册”页签填写用户名、邮箱、姓名和密码后完成注册；
5. 登录成功后，系统会跳转到首页或原先要访问的页面。

### 3.2 首页仪表盘

首页路由为：`/`。

首页主要作用是快速展示当前会议的统计信息和最近会议内容，便于用户进入系统后先了解整体情况，再进入具体功能模块。

首页一般可以用于：

- 查看会议概况；
- 快速进入会议列表；
- 快速进入任务中心；
- 查看最近会议。

## 4 会议管理功能

### 4.1 会议列表

会议列表页面路由为：`/meetings`。

在该页面中，用户可以完成以下操作：

- 查看系统中的会议；
- 按状态筛选会议；
- 按关键词搜索会议；
- 分页浏览会议；
- 新建会议；
- 删除会议。

### 4.2 新建会议

在会议列表页点击“新建会议”按钮后，会弹出创建表单。表单中可以填写：

- 会议标题；
- 会议描述；
- 所属团队；
- 组织者；
- 开始时间；
- 结束时间；
- 地点；
- 参会人员。

操作步骤如下：

1. 进入会议列表页；
2. 点击“新建会议”；
3. 按实际情况填写会议信息；
4. 点击“创建”；
5. 创建成功后返回会议列表。

### 4.3 删除会议

在会议列表中，用户可以点击会议右侧的删除按钮删除会议。系统会先弹出确认框，确认后才会真正删除数据。

该操作更适合测试数据清理，正式演示时建议谨慎使用。

## 5 会议详情工作台

### 5.1 页面简介

会议详情页面路由为：`/meetings/:id`。

这是系统中最核心的页面，用于集中管理某一场会议的全部内容。页面中主要包含以下区域：

- 会议信息区；
- 统计概览区；
- 音频文件区；
- 在线录音区；
- 摘要区；
- 转写区；
- 任务区；
- 参会人区。

页面顶部还提供“刷新”“清空会议内容”“生成分享链接”“邮件分发”“新建任务”等按钮。

### 5.2 上传音频并转写

适用于已经有录音文件的会议场景。操作步骤如下：

1. 进入会议详情页；
2. 选择本地音频文件；
3. 上传音频；
4. 等待系统完成转写；
5. 在转写区查看识别结果。

### 5.3 在线录音

在线录音适用于实时记录会议内容。操作流程如下：

1. 进入会议详情页；
2. 点击“开始录音”；
3. 浏览器弹出麦克风权限提示时点击允许；
4. 录音中可以点击“暂停录音”；
5. 需要继续时点击“继续录音”；
6. 会议结束后点击“停止并转写”；
7. 系统自动完成上传、转写和刷新。

注意事项如下：

- 录音依赖浏览器支持麦克风权限；
- 当前主要面向麦克风输入；
- 如果权限被拒绝，系统会提示无法访问麦克风。

### 5.4 查看摘要与转写

会议详情页中还可以直接查看系统生成的摘要与转写内容：

- 摘要区用于展示会议要点、决议和待办事项；
- 转写区用于展示语音识别后的完整文本内容。

通过摘要和转写，用户可以更方便地回顾会议过程并提取重点。

### 5.5 任务管理

会议详情页中的任务区域支持以下操作：

- 查看任务；
- 新建任务；
- 编辑任务；
- 修改任务状态。

如果系统已经从会议内容中提取出了任务，用户也可以继续补充责任人、优先级和截止时间。

### 5.6 分享链接与邮件分发

当会议已经生成摘要后，可以：

- 生成分享链接；
- 复制分享地址；
- 通过邮件分发会议结果。

分享页面路由形式为：`/shared/meetings/:token`，主要用于只读查看会议结果。

### 5.7 清空会议内容

系统支持选择性清空会议内容，可清除：

- 转写；
- 任务；
- 摘要；
- 音频；
- 会议状态。

这个功能主要用于调试和重复演示。

## 6 任务中心功能

任务中心页面路由为：`/tasks`。

这个页面适合统一查看和管理系统中的全部任务。用户可以在这里进行以下操作：

- 按状态筛选任务；
- 按优先级筛选任务；
- 输入关键词检索任务；
- 按创建时间或截止时间排序；
- 直接修改任务状态；
- 编辑任务信息；
- 跳转查看任务所属会议。

推荐使用流程如下：

1. 进入任务中心；
2. 根据任务状态或优先级筛选需要关注的任务；
3. 检查是否有高优先级或逾期任务；
4. 直接更新任务状态；
5. 如需查看会议上下文，再跳转到会议详情页。

## 7 团队与邀请管理

### 7.1 团队列表

团队页面路由为：`/teams`。

页面一般分为“我的团队”和“可发现团队”两个区域，用户可以在这里查看已加入的团队，也可以浏览公开团队并加入。

### 7.2 创建团队

创建团队页面路由为：`/teams/create`。

如果需要按照课程小组、项目小组或实验小组来组织会议，可以先创建团队，再把成员和会议逐步管理起来。

团队详情页路由为：`/teams/:id`。

在该页面中，用户可以：

- 查看团队基本信息；
- 邀请成员；
- 生成邀请链接；
- 调整成员角色；
- 移除成员；
- 删除团队（通常仅限 owner）。

### 7.4 邀请管理

邀请列表页面路由为：`/invitations`。

用户可以在页面中查看收到的团队邀请，并选择接受或拒绝。如果是通过邀请链接直接访问，则对应页面路由为：`/invite/:token`。

## 8 共享会议、AI 助手与热词管理

## 10 常见问题与处理方法

### 10.1 后端提示 `Form data requires python-multipart`

原因：后端上传相关依赖没有安装完整。

解决方法：

```bash
python -m pip install -r backend/requirements.txt
```

### 10.2 前端安装依赖时出现权限错误

原因：Windows 环境下 npm 默认缓存目录可能存在权限问题。

解决方法：

```bash
npm --prefix frontend install --cache "D:\SmartMeeting\.npm-cache"
```

### 10.3 浏览器无法录音

可能原因包括：

- 浏览器没有授权麦克风；
- 浏览器不支持 `MediaRecorder`；
- 麦克风设备本身不可用。

建议处理步骤：

1. 检查浏览器权限设置；
2. 确认网站已被允许访问麦克风；
3. 尝试更换浏览器；
4. 如果仍无法录音，可以先改用音频上传方式。

### 10.4 没有配置 AI 服务还能否使用

可以。即使没有真实的 LLM 或 Whisper 服务，系统依然可以运行，只是摘要、任务提取或转写结果会使用回退逻辑，效果和正式环境相比会有差异。

### 10.5 分享链接无法生成

根据当前页面逻辑，会议未生成摘要时，分享按钮会被禁用。因此应先完成摘要生成，再执行分享操作。
