# 一、立项摘要

SmartMeeting是一款**面向中小企业与远程协作团队**的智能会议SaaS产品，聚焦解决当前企业会议场景下记录效率低下、会后执行断层、跨团队协同薄弱的核心痛点。项目依托成熟的AI语音转写、大模型纪要生成与智能任务提取技术，打造“会前-会中-会后”全流程闭环的会议协作工具，实现会议信息自动结构化、任务执行全程可追踪。

本项目由**张航星单人独立完成全流程开发**，兼顾产品设计、全栈开发、测试运维全链路工作，结合精简开发范围、聚焦核心刚需的原则，计划**6周内完成可上线MVP版本交付**，上线后3个月内完成首批种子用户验证与产品轻量迭代。通过轻量化、低门槛的产品形态，快速切入中小团队会议协作市场，打造高效便捷的智能会议协作解决方案，全面提升企业会议效率与执行落地能力。

# 二、项目背景与市场机会

## 2.1 业务背景

随着远程办公、跨地域协作成为中小企业常态化工作模式，企业线上线下会议频次持续攀升，但现有会议协作流程普遍存在信息沉淀不足、执行闭环缺失的问题，具体痛点突出：

- **人工记录成本高、质量不稳定**：传统人工手写或打字记录会议内容，耗时耗力，且容易遗漏关键决策、行动项与核心结论，不同记录人员输出的纪要标准不一，后续复盘难度大；

- **会后执行断层严重**：会议结论与待办任务分散在聊天记录、文档、备忘录中，无统一管理入口，任务跟进不及时、责任划分不清晰，导致会议成果无法有效落地；

- **现有工具功能单一**：市面上多数会议工具仅聚焦音视频沟通或单纯语音转写，缺少“会议记录-纪要生成-任务拆解-跟进督办”的一体化链路，无法满足团队高效协作需求。

## 2.2 市场机会

当前技术环境与市场需求双重利好，智能会议系统具备清晰的落地与发展空间：

- **技术成熟度支撑**：AI大模型、语音识别技术日趋完善，语音转写、文本摘要、信息提取等能力商业化落地可行性大幅提升，可实现会议场景的全流程自动化；

- **目标市场需求明确**：中小企业与远程团队预算有限、人员精简，对**轻量易用、低门槛、快速上线**的SaaS协作工具有刚性需求，排斥复杂笨重、部署成本高的企业级软件；

- **差异化切入优势**：以“AI转写+智能纪要+任务追踪”的高频刚需组合为核心卖点，区别于单纯的会议记录工具，打造“会议到执行”的完整闭环，更容易培养用户使用习惯，形成稳定续费与用户留存基础。

# 三、项目目标与成功指标（KPI）

## 3.1 核心项目目标

1. 搭建适配单人开发节奏、可直接商用的智能会议系统MVP版本，覆盖会议音频处理、纪要生成、任务管理核心流程，满足基础会议协作需求；

2. 实现会议纪要产出效率**提升70%以上**，替代传统人工手动整理模式；

3. 打通会议任务从提取到落地的闭环，将会议任务可追踪率提升至**90%以上**。

## 3.2 关键成功指标（KPI）

|---|---|---|---|
|效率指标|单次会议纪要生成时间|≤3分钟|每周|

|质量指标|会议任务提取准确率|≥85%|每周|

# 五、产品范围定义

## 5.1 MVP核心范围（In Scope）

本项目为单人独立开发，为保障6周内高效交付可用MVP，严格聚焦核心刚需功能，砍掉非必要分支需求，首期仅覆盖最核心的会议智能处理模块，降低开发复杂度，具体包含以下内容：

- 会议音频上传与离线语音转写功能（暂不开发实时转写，减少技术复杂度与开发量）；

- AI自动生成结构化会议纪要，支持基础重点内容标注；

- 会议核心决策、待办任务智能提取，支持手动完善负责人信息；

- 基础任务分配、状态标记、简单到期提醒功能；

- 会议记录单账号存储与基础链接分享功能。

## 5.2 非MVP范围（Out of Scope）

结合单人开发、6周周期的实际情况，以下功能暂不纳入首期开发，后续根据用户反馈与个人时间精力逐步迭代，避免延期风险：

- 实时语音转写、视频会议SDK原生集成；

- 企业级SSO/LDAP统一登录、精细化团队权限管控；

- 复杂BI数据报表、高级数据分析、多维度统计功能；

- 私有化本地部署、多语言深度适配、第三方日历/IM工具对接。

# 七、技术方案与架构设计

## 7.1 核心技术栈选型

|技术层级|具体技术选型|
|---|---|
|前端开发|Vue 3 + TypeScript + Element Plus|
|后端开发|FastAPI（Python）|
|数据存储|MySQL|
|AI核心能力|Whisper语音识别模型 + 大语言模型（LLM）|
|部署运维|Docker + Nginx|
## 7.2 系统整体架构

```text
┌──────────────┐    ┌──────────────┐    ┌──────────────┐
│   Frontend   │───▶│   FastAPI    │───▶│    MySQL     │
│  Vue3 + TS   │    │  API/Service │    │   Storage    │
└──────────────┘    └──────┬───────┘    └──────────────┘
                           │
                  ┌────────┴────────┐
                  │  AI Pipeline    │
                  │ Whisper + LLM   │
                  └─────────────────┘
```

## 7.3 非功能性需求

- **可用性**：核心API接口可用性≥99.5%，保障系统稳定运行；

- **性能**：30分钟时长会议，3-5分钟内完成完整纪要与任务提取；

- **安全性**：数据传输全程采用HTTPS加密，敏感配置通过环境变量管理，杜绝信息泄露；

- **可观测性**：搭建核心链路日志、错误实时告警、接口调用耗时监控体系，方便单人运维排查问题。

## 9.2 单人进度管控机制

- 周进度复盘机制：每周日梳理本周完成内容、剩余任务与待解决问题，制定下周每日精准工作计划，严控开发节奏，杜绝延期；

- 每日任务拆解：将大模块拆分为每日可完成的小任务，当日任务当日清，遇到技术卡点及时记录并优先攻克，不拖延至后续周期；

- 阶段性自测机制：每个模块开发完成后立即自测，提前发现并修复问题，避免后期集中测试工作量过大；上线前完成全流程回归测试，做好部署与回滚预案。
