# 一、立项摘要

本项目**名义以团队形式立项递交**，符合课程项目提交规范要求，实际由张航星单人独立完成全流程工作，兼顾产品设计、全栈开发、测试运维、初期运营全链路职责，全程对标互联网优秀SaaS团队开发标准，摒弃无序开发模式，依托标准化流程推进。项目聚焦中小团队会议协作痛点，计划**6周内完成可上线MVP版本交付**，上线后3个月内完成首批种子用户验证与产品轻量迭代。

SmartMeeting是一款**AI驱动的轻量化智能会议SaaS产品**，核心作用是替代传统人工会议记录，打通“会议记录-纪要生成-任务拆解-跟进督办”全流程闭环，解决中小团队会议记录效率低、会后执行断层、跨团队协同薄弱的核心问题，降低团队沟通成本，提升会议成果落地率。产品主打轻量化、易上手、低成本，精准切入5-50人规模中小企业、远程协作团队细分市场，依托成熟AI语音转写与大模型技术，快速实现核心价值落地。

# 二、项目背景与需求深度调研

## 2.1 业务背景与市场现状

随着远程办公、跨地域协作成为常态化工作模式，各类团队会议频次持续攀升，但现有会议协作流程存在普遍痛点，市场缺口明显：传统人工记录会议内容耗时耗力，易遗漏关键决策与待办事项，纪要标准不统一；会后任务分散在各类文档、聊天记录中，无统一管理入口，任务跟进不及时、责任不清晰，导致会议成果难以落地；市面上现有工具要么仅支持音视频沟通，要么仅具备单一语音转写功能，缺少从会议到执行的一体化闭环能力，且大型企业级软件功能繁杂、成本偏高，不适配中小团队轻量化需求。

同时，AI大模型与语音识别技术日趋成熟，商业化落地成本降低，为轻量化智能会议工具提供了技术支撑；中小团队与远程协作群体规模庞大，对低成本、易上手的协作工具有刚性需求，市场具备清晰的发展空间。

## 2.2 核心需求调研要点（全维度明确）

本次需求调研采用**用户访谈+竞品对标+场景复盘**的标准化方法，对标飞书纪要、腾讯会议智能纪要、Notion AI等优质产品，结合真实办公场景梳理核心需求，为后续需求规划书提供完整依据，具体要点如下：

### 2.2.1 软件核心作用

- 替代人工会议记录，将传统30分钟以上的纪要整理时间压缩至3分钟内，大幅提升记录效率；

- 自动结构化沉淀会议内容，智能提取核心决策与待办任务，实现会议信息可追溯、可复用；

- 搭建任务跟踪闭环，明确任务负责人与完成时限，解决会议“重过程、轻落地”的行业通病；

- 提供轻量化协作入口，支持会议内容共享，降低跨成员协同成本，无需复杂部署即可使用。

### 2.2.2 目标用户群体

- **核心群体**：5-50人规模中小企业、远程兼职团队、小型项目组，预算有限、人员精简，排斥复杂企业软件；

- **次要群体**：自由职业者、高校课题小组、初创创业团队，有高频小型会议、复盘需求，追求便捷易用；

- **用户角色细分**：团队管理者、项目经理、普通参会成员，三类角色需求各有侧重，覆盖全场景使用。

### 2.2.3 核心功能需求

- 基础层：会议音频上传、离线AI语音转写，支持常见音频格式，转写准确率满足日常办公需求；

- 核心层：自动生成结构化会议纪要，支持重点内容标注，智能提取会议决策、待办任务；

- 协作层：基础任务分配、状态标记、简易提醒，会议记录单账号存储与链接分享；

- 体验层：界面简洁易操作，无需学习成本，响应速度快，适配各类浏览器使用。

## 2.3 优秀软件开发团队经验借鉴

为保障项目高效落地、规避单人开发常见误区，深度调研飞书、钉钉轻应用、小众优质SaaS团队的开发方法论，提炼适配本项目的实操经验，全程遵循执行：

- **需求管控经验**：严格坚守MVP原则，杜绝需求蔓延，优先开发P0核心刚需功能，非核心功能全部延后迭代，不追求初期完美，先跑通核心业务闭环，这是小团队/单人高效交付的核心准则；

- **流程规范经验**：采用敏捷开发模式，拆分短周期迭代节点，每周设定明确里程碑，每日拆解小任务，阶段性复盘验收，避免后期集中返工；拒绝“边做边改”，前期敲定需求细节，减少中途变更；

- **技术选型经验**：优先选用成熟、生态完善、上手快速的技术框架，借助开源组件与第三方成熟API降低自研复杂度，不重复造轮子；采用模块化、前后端分离架构，方便后续迭代拓展；

- **质量管控经验**：建立标准化自测流程，模块开发完成立即测试，核心流程反复验证，上线前全链路回归测试，弥补单人无交叉测试的短板；

- **项目管控经验**：做好时间规划，预留缓冲周期应对突发问题，合理分配精力，避免单一模块耗时过长导致整体延期。

# 五、产品范围定义

## 5.1 MVP核心范围（In Scope）

结合单人开发、6周周期的实际情况，严格聚焦核心刚需，砍掉非必要功能，降低开发复杂度，首期MVP仅覆盖以下核心内容，保障按期交付：

- 会议音频上传与离线语音转写功能（暂不开发实时转写，大幅减少技术与工作量）；

- AI自动生成结构化会议纪要，支持基础重点内容标注与手动编辑；

- 会议核心决策、待办任务智能提取，支持手动完善负责人、时限信息；

- 基础任务分配、状态标记、简单到期提醒功能；

- 会议记录单账号存储与基础链接分享功能。

## 5.2 非MVP范围（Out of Scope）

以下功能暂不纳入首期开发，后续根据用户反馈、时间精力逐步迭代，避免需求蔓延导致延期：

- 实时语音转写、视频会议SDK原生集成功能；

- 企业级SSO/LDAP统一登录、精细化团队权限管控；

- 复杂BI数据报表、高级数据分析、多维度统计功能；

- 私有化本地部署、多语言深度适配、第三方日历/IM工具对接。

# 七、技术方案与架构设计

## 7.1 核心技术栈选型

技术选型严格遵循**成熟、易用、轻量**原则，适配单人开发节奏，降低运维难度，具体如下：

|技术层级|具体选型|
|---|---|
|前端开发|Vue 3 + TypeScript + Element Plus|
|后端开发|FastAPI（Python）|
|数据存储|MySQL|
|AI核心能力|Whisper语音识别模型 + 大语言模型（LLM）|
|部署运维|Docker + Nginx|
## 7.2 系统整体架构

```text
┌──────────────┐    ┌──────────────┐    ┌──────────────┐
│   Frontend   │───▶│   FastAPI    │───▶│    MySQL     │
│  Vue3 + TS   │    │  API/Service │    │   Storage    │
└──────────────┘    └──────┬───────┘    └──────────────┘
                           │
                  ┌────────┴────────┐
                  │  AI Pipeline    │
                  │ Whisper + LLM   │
                  └─────────────────┘
```

## 7.3 非功能性需求

- **可用性**：核心API接口可用性≥99.5%，保障系统稳定运行；

- **性能**：30分钟时长会议，3-5分钟内完成完整纪要与任务提取；

- **安全性**：数据传输全程HTTPS加密，敏感配置通过环境变量管理，杜绝信息泄露；

- **可观测性**：搭建核心链路日志、错误告警、耗时监控，方便单人运维排查问题。

# 八、标准化软件开发全流程

本项目全程对标互联网优秀团队，执行**标准化、全流程、可追溯**的开发流程，摒弃随意开发模式，共分为六大核心阶段，每阶段明确目标与产出物，为后续需求规划书提供完整框架，具体流程如下：

## 8.1 需求调研与分析阶段

核心工作：用户调研、竞品分析、痛点梳理、需求清单梳理、优先级划分；产出物：完整需求调研文档、核心需求要点清单，明确产品功能边界，为后续开发定调。

## 8.2 产品规划与设计阶段

核心工作：产品定位确认、MVP范围划定、功能模块设计、版本路线规划；产出物：产品功能规划书、MVP需求文档，明确可做与不可做功能，杜绝需求变更。

## 8.3 技术设计与方案验证阶段

核心工作：技术栈选型、系统架构设计、数据库表设计、接口方案设计、AI接口预调试；产出物：技术设计文档、数据库结构图，提前验证技术可行性，规避后期技术卡点。

## 8.4 敏捷开发与迭代阶段

核心工作：模块化开发、每日任务推进、每周里程碑验收、阶段性自测；遵循“先核心后次要”原则，先完成P0功能，再开发P1功能，每日闭环任务，每周复盘进度。

## 8.5 测试修复与优化阶段

核心工作：制定测试用例、功能测试、性能压测、BUG修复、全流程回归测试、体验优化；单人完成全维度测试，确保无阻塞性问题，达到上线标准。

## 8.6 上线部署与运营迭代阶段

核心工作：云端部署、灰度试用、用户反馈收集、小版本迭代、运营落地；产出物：部署手册、运维文档，启动种子用户运营，持续优化产品。
