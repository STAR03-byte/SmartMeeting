# SmartMeeting 软件使用说明书

## 1. 编写说明

这份说明书是结合 SmartMeeting 当前已经完成的功能整理出来的，主要用于课程项目演示、验收和后续使用参考。写这份文档的时候，我尽量按照实际项目里的页面、脚本和运行方式来写，避免只写概念、不写操作。

本文档重点说明以下内容：

- 软件的运行环境和安装方式；
- 系统的启动方法；
- 各个主要功能模块的使用流程；
- 常见问题及处理方法；
- 后续做图文版说明书时的截图建议。

## 2. 软件简介

SmartMeeting 是一个面向会议记录和任务整理的智能会议系统，项目技术栈为 FastAPI + Vue 3 + TypeScript + MySQL。系统主要解决的问题是：把会议过程中的音频、转写、摘要和任务集中到一个页面里管理，减少人工整理会议纪要的重复工作。

目前系统已经实现的核心功能包括：

- 用户登录与注册；
- 会议创建、查询、筛选、删除；
- 上传音频进行转写；
- 在线录音并在结束后自动转写；
- 自动生成会议摘要；
- 从会议内容中整理任务；
- 任务中心统一查看和更新任务；
- 团队管理与邀请；
- 会议分享；
- AI 助手和热词管理。

## 3. 适用对象与使用场景

### 3.1 适用对象

这套系统适合以下用户使用：

- 需要整理会议纪要的小组成员；
- 需要跟踪任务分配情况的项目负责人；
- 需要做课程演示、系统验收或毕业设计答辩的老师和同学。

### 3.2 典型使用场景

1. **日常组会记录**  
   小组成员在开会时直接使用在线录音，会议结束后自动得到转写和摘要。

2. **已有录音整理**  
   如果会议已经提前录好了音频，也可以直接上传到系统中，再进行转写和摘要生成。

3. **任务跟踪**  
   项目负责人可以把会议中产生的任务统一放到任务中心，再持续更新状态。

4. **团队协作**  
   团队管理员可以邀请其他成员加入团队，共同查看会议和任务。

5. **会议结果分享**  
   当会议摘要整理完成后，可以生成只读分享链接发给没有直接参与系统操作的人查看。

## 4. 运行环境

根据当前项目中的启动脚本、README 和前端联调文档，建议的运行环境如下：

| 项目 | 建议环境 |
|------|----------|
| 操作系统 | Windows 10/11、Linux、macOS |
| Python | 3.8 及以上 |
| Node.js | 18 及以上 |
| 数据库 | MySQL 8.0 |
| 浏览器 | Chrome、Edge 等支持麦克风权限与 `MediaRecorder` 的现代浏览器 |
| 可选能力 | Docker / Docker Compose、Whisper、LLM 服务 |

说明：

- 如果没有配置真实的 LLM 服务，系统会使用规则回退方式生成摘要和任务；
- 如果没有配置真实的 Whisper 语音识别，系统会使用模拟转写或回退逻辑；
- 在线录音依赖浏览器麦克风权限，因此建议优先使用桌面版 Chrome 或 Edge 进行演示。

## 5. 安装与启动

### 5.1 方式一：使用 Docker 启动（推荐）

在项目根目录执行：

```bash
docker compose up --build
```

启动完成后，默认访问地址如下：

- 前端：`http://localhost:5174`
- 后端：`http://localhost:8000`
- MySQL：`localhost:3307`

这种方式适合第一次完整体验系统，因为前后端和数据库会一起启动，环境相对统一。

### 5.2 方式二：本地开发一键启动（推荐用于调试）

在项目根目录执行：

```bash
npm run dev
```

这个命令会自动完成以下工作：

- 检查前后端依赖是否已经准备好；
- 如有缺失，调用 `scripts/dev/bootstrap.py` 安装依赖；
- 启动后端服务；
- 自动选择一个可用的前端端口并启动前端。

默认情况下：

- 后端地址为 `http://127.0.0.1:8000`
- 前端地址优先为 `http://127.0.0.1:5173`

### 5.3 方式三：手动分别启动前后端

如果需要分开调试，也可以手动启动。

后端：

```bash
python -m pip install -r backend/requirements.txt
python -m uvicorn backend.main:app --reload
```

前端：

```bash
npm --prefix frontend install --cache "D:\SmartMeeting\.npm-cache"
npm --prefix frontend run dev
```

如果本地后端不是运行在默认的 `8000` 端口，可以在启动前设置：

```bash
set SMARTMEETING_DEV_BACKEND_URL=http://127.0.0.1:8888
```

### 5.4 数据库初始化

如果是第一次搭建数据库，可以执行：

```bash
mysql -u <user> -p < scripts/db/run_all.sql
```

执行完成后，可以再使用以下脚本检查数据库状态：

```bash
mysql -u <user> -p < scripts/db/check_health.sql
```

### 5.5 AI 与语音能力说明

系统支持两类增强能力：

1. **会议摘要与任务提取**：依赖 LLM 配置；
2. **语音转写**：依赖 Whisper 配置。

如果没有配置相关环境变量，系统依然可以运行，只是生成结果会使用回退逻辑，适合课程项目演示和基础联调。

## 6. 系统访问与基本使用

### 6.1 登录与注册

系统登录页路由为：`/login`

登录页支持两个入口：

- 登录
- 注册

使用方法如下：

1. 打开系统首页，如果当前没有登录，会自动跳转到登录页；
2. 如果已有账号，输入用户名和密码后点击“登录”；
3. 如果没有账号，可以切换到“注册”标签页，填写用户名、邮箱、姓名和密码后完成注册；
4. 登录成功后，系统会自动跳转回原本想访问的页面，或者进入首页。

> 截图建议：此处可放“登录页整体界面截图”，并标注“登录”“注册”两个页签以及用户名、密码输入区域。

### 6.2 首页仪表盘

系统首页路由为：`/`

首页主要用于快速了解当前会议情况，常见内容包括：

- 会议统计信息；
- 最近会议；
- 快捷入口。

适合的使用方式是：先在首页查看整体情况，再根据需要进入会议列表、任务中心或某一场具体会议。

## 7. 功能模块操作说明

### 7.1 会议管理

#### 7.1.1 进入会议列表

会议列表页面路由为：`/meetings`

这个页面支持以下操作：

- 查看会议列表；
- 按状态筛选会议；
- 按关键词搜索会议；
- 分页浏览；
- 新建会议；
- 删除会议。

#### 7.1.2 新建会议

在会议列表页点击“新建会议”后，会弹出创建表单。可以填写的内容包括：

- 会议标题；
- 会议描述；
- 所属团队；
- 组织者；
- 计划开始时间；
- 计划结束时间；
- 地点；
- 参会人员。

操作步骤如下：

1. 进入 `/meetings` 页面；
2. 点击右上角“新建会议”；
3. 按实际情况填写会议信息；
4. 点击“创建”；
5. 创建成功后，系统会刷新会议列表。

适用场景：会前先把会议基本信息录入系统，方便后续上传音频、录音和整理任务。

> 截图建议：放“新建会议弹窗截图”，标注标题、团队、组织者、开始时间等关键字段。

#### 7.1.3 删除会议

如果某条会议记录不再需要，可以在会议列表右侧点击“删除”。系统会二次确认，确认后才真正删除。

这个操作适合清理测试数据，但正式演示时建议谨慎使用。

### 7.2 会议详情工作台

会议详情页面路由为：`/meetings/:id`

这是本系统最核心的页面，主要把同一场会议相关的数据集中展示在一个工作台里。页面主要包含以下区域：

- 会议信息区；
- 统计概览；
- 音频文件区；
- 在线录音区；
- 摘要区；
- 转写区；
- 任务区；
- 参会人区。

页面顶部还提供以下常用按钮：

- 刷新；
- 清空会议内容；
- 生成分享链接；
- 邮件分发；
- 新建任务。

#### 7.2.1 上传音频并转写

适用于已经有会议录音文件的情况。

基本流程如下：

1. 进入某场会议详情页；
2. 在音频相关区域选择本地音频文件；
3. 上传成功后触发转写；
4. 系统完成处理后，在转写面板中可以看到结果；
5. 如果后处理成功，还可以继续生成摘要和任务。

适用场景：线下会议已经录音，想在会后统一整理内容。

#### 7.2.2 在线录音

在线录音是系统中比较方便的一个功能，适用于实时记录会议。

操作步骤如下：

1. 进入会议详情页；
2. 点击“开始录音”；
3. 浏览器第一次使用时会请求麦克风权限，点击允许；
4. 录音过程中可以点击“暂停录音”；
5. 如需继续，点击“继续录音”；
6. 会议结束后点击“停止并转写”；
7. 系统会自动上传录音、执行转写并刷新页面中的相关数据。

需要注意：

- 该功能依赖浏览器支持麦克风权限；
- 当前主要面向麦克风输入，不包含系统内录音；
- 如果浏览器拒绝麦克风权限，系统会提示无法访问麦克风。

> 截图建议：可以分成两张图，一张展示“开始录音”前的状态，一张展示“录音中/暂停中”的状态。

#### 7.2.3 查看会议摘要

摘要区用于展示系统生成的会议纪要结果。摘要通常会包含：

- 会议要点；
- 决议内容；
- 待办事项。

如果系统拿到了结构化结果，页面会按结构展示；如果没有结构化结果，也会显示普通文本摘要。

适用场景：开完会后快速回顾重点，不用再从长篇转写里逐句查找。

#### 7.2.4 查看会议转写

转写面板用于展示系统识别出来的文字内容。一般可以用来：

- 回看会议中的原始讨论内容；
- 辅助检查摘要是否准确；
- 在后续写任务或纪要时引用原话。

#### 7.2.5 管理会议任务

会议详情页中的任务区支持：

- 查看任务；
- 新建任务；
- 编辑任务；
- 修改任务状态。

如果系统已经根据会议内容提取出了任务，用户可以在这里继续补充责任人、优先级和截止时间。

适用场景：会后马上把“谁来做、什么时候完成”补全，减少后续遗漏。

#### 7.2.6 管理参会人员

会议详情页也支持查看和维护参会人员信息，方便整理会议参与情况，并在邮件分发或协作时使用。

#### 7.2.7 生成分享链接

当会议已经生成摘要后，可以点击“生成分享链接”。

使用流程如下：

1. 确保当前会议已经有摘要；
2. 点击“生成分享链接”；
3. 系统会自动复制链接；
4. 接收者打开链接后，可以通过只读页面查看摘要、转写和任务。

分享页面路由形式为：`/shared/meetings/:token`

这个功能适合把会议结果发给不直接参与系统操作的老师、同学或组员。

#### 7.2.8 邮件分发

如果会议参与人中已经有邮箱信息，可以点击“邮件分发”。系统会根据分享链接和摘要内容，帮助打开本地邮件草稿，便于快速发送会议结果。

#### 7.2.9 清空会议内容

为了便于调试和重复测试，系统支持选择性清空会议内容。可以清除的内容包括：

- 转写；
- 任务；
- 摘要；
- 音频；
- 会议状态。

这个功能适合课程演示前重置测试数据，但正式记录重要会议时不建议随意使用。

### 7.3 任务中心

任务中心页面路由为：`/tasks`

这个页面适合统一查看系统中的所有任务，而不是只看某一场会议中的任务。页面支持：

- 按任务状态筛选；
- 按优先级筛选；
- 按关键词搜索；
- 按创建时间或截止时间排序；
- 直接修改任务状态；
- 编辑任务详细信息；
- 跳转查看任务对应的会议。

操作建议：

1. 先按状态或优先级筛选出需要处理的任务；
2. 检查是否有逾期任务或高优先级任务；
3. 直接在任务列表中更新状态；
4. 如需查看上下文，点击会议链接跳回会议详情页。

适用场景：周会后统一检查本周待完成事项。

> 截图建议：放“任务中心截图”，标注筛选区、状态切换区和会议跳转入口。

### 7.4 团队管理

#### 7.4.1 团队列表

团队页面路由为：`/teams`

页面一般分为：

- 我的团队；
- 可发现团队。

在这里可以查看已加入的团队，也可以浏览公开团队并加入。

#### 7.4.2 创建团队

创建团队页面路由为：`/teams/create`

如果用户希望按项目组、课程组或实验组来管理会议，可以先创建团队，再把会议和成员逐步组织起来。

#### 7.4.3 团队详情与成员管理

团队详情页面路由为：`/teams/:id`

在团队详情页中，用户可以进行以下操作：

- 查看团队基本信息；
- 邀请新成员；
- 生成邀请链接；
- 调整成员角色；
- 移除成员；
- 删除团队（通常仅限 owner）。

适用场景：课程小组成员变动时，统一维护团队成员关系。

### 7.5 邀请管理

邀请列表页面路由为：`/invitations`

用户可以在这里看到自己收到的团队邀请，并进行：

- 接受邀请；
- 拒绝邀请。

如果是通过邀请链接直接进入系统，对应页面路由为：`/invite/:token`，用户可以在该页面完成加入团队操作。

### 7.6 共享会议页面

共享会议页面路由为：`/shared/meetings/:token`

这个页面主要用于只读查看会议内容，常见展示内容包括：

- 会议摘要；
- 会议转写；
- 会议任务。

适用场景：把会议成果发给老师查看，或者发给只需要阅读结果、不需要编辑的人。

### 7.7 AI 助手

AI 助手页面路由为：`/ai-assistant`

这个页面支持以对话方式辅助处理会议和任务内容。根据当前代码，AI 助手支持：

- 多会话；
- 选择会议或任务作为上下文；
- 生成任务草稿。

适用场景：当用户想根据已有会议内容进一步整理问题、补写任务说明或辅助总结时，可以使用这个功能。

### 7.8 热词管理

热词管理页面路由为：`/hotwords`

这个页面用于管理转写过程中的热词，常见操作包括：

- 新增热词；
- 查看热词列表；
- 删除热词。

适用场景：如果会议中经常出现项目名、缩写词、专业术语或人名，可以提前加入热词，帮助转写结果更接近真实表达。

## 8. 推荐使用流程

如果是第一次使用系统，可以按照下面的顺序操作：

1. 启动系统；
2. 注册或登录；
3. 创建一个团队（可选）；
4. 创建一场会议；
5. 进入会议详情页；
6. 选择“上传音频”或者“在线录音”；
7. 等待系统完成转写；
8. 查看摘要结果；
9. 检查并补充任务；
10. 需要时生成分享链接或邮件分发；
11. 最后进入任务中心持续跟踪任务状态。

这个流程比较适合课程展示，因为能够把系统主要功能完整串起来。

## 9. 常见问题与处理方法

### 9.1 后端提示 `Form data requires python-multipart`

原因：后端上传相关依赖没有安装完整。

处理方法：

```bash
python -m pip install -r backend/requirements.txt
```

### 9.2 前端安装依赖时出现权限错误（例如 EPERM）

原因：Windows 环境下 npm 默认缓存目录可能存在权限问题。

处理方法：

```bash
npm --prefix frontend install --cache "D:\SmartMeeting\.npm-cache"
```

### 9.3 浏览器无法录音

可能原因：

- 浏览器未授权麦克风；
- 当前浏览器不支持 `MediaRecorder`；
- 麦克风设备本身不可用。

处理建议：

1. 检查浏览器地址栏附近的权限提示；
2. 确认已经允许网站访问麦克风；
3. 更换 Chrome 或 Edge 重试；
4. 如果还是不行，可以先使用“上传音频”功能替代在线录音。

### 9.4 前端类型检查报第三方库类型问题

当前项目已经在前端配置中启用了相应规避策略，常见第三方类型噪音通常不会影响系统基本运行。如果仍然遇到问题，建议先确认依赖版本是否和项目当前锁定版本一致。

### 9.5 没有配置 AI 服务还能不能使用

可以使用。系统在没有真实 LLM 或 Whisper 配置时，仍然可以进入页面、管理会议和任务，只是摘要、任务提取或转写结果会使用回退逻辑，效果与正式部署环境相比会有差异。

### 9.6 分享链接为什么不能生成

根据当前页面逻辑，会议还没有摘要时，生成分享链接按钮会被禁用。因此需要先完成摘要生成，再执行分享。
